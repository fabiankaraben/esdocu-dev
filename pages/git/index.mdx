# ¿Qué es Git?

¿Alguna vez has perdido horas de trabajo porque borraste accidentalmente un archivo u olvidaste guardarlo? ¿O tal vez querías volver a una versión anterior de un archivo pero no pudiste porque no se guardó el historial de cambios?

Estos son los problemas que resuelven los **Sistemas de Control de Versiones**. Te permiten guardar el historial de cambios y volver a cualquier versión anterior de un archivo. Además de eso, te permiten compartir el historial de cambios con otras personas.

![Git](/assets/git/git-home-01.jpg)

**Git** es el estándar de facto para el control de versiones. Se utiliza en casi todos los proyectos de software. Si eres desarrollador de software, necesitarás conocer Git. Pero también será útil para cualquiera que trabaje con archivos de texto.

La experiencia de varios fundadores de startups relata que, entre otras cosas, sabían cómo usar Git, y esto les ayudó a comprender mejor el proceso de desarrollo y redujo la fricción al trabajar con el equipo de desarrollo.

Por ejemplo, podían extraer los últimos cambios del repositorio y probarlos en su máquina local. O podían cambiar a una rama creada para una característica particular y obtener una vista previa allí.

En general, les dio una mayor comprensión de lo que estaba pasando con el proyecto y les ayudó a tomar mejores decisiones.

## ¿Por qué utilizar la línea de comandos?

Git tiene muchas aplicaciones con interfaces gráficas de usuario (GUI). Algunos de ellos son muy buenos. Entonces, **¿por qué utilizar la línea de comando?**

Creo que usar Git desde la línea de comandos es la mejor manera de aprenderlo. También es la forma más flexible de utilizarlo. No estarás limitado a una aplicación GUI en particular o a un complemento de edición. Podrás utilizar Git desde un servidor remoto, desde una máquina virtual o desde un contenedor Docker.

Otro beneficio es que aprenderás el "lenguaje" que utiliza Git y no el "lenguaje" de una aplicación GUI en particular. Algunas aplicaciones GUI ocultan la complejidad de Git. Funciona muy bien hasta que algo sale mal. Al usar Git desde la línea de comandos, aprenderás los términos y conceptos que usa Git. Esto te ayudará a comunicarte con otros desarrolladores y obtener ayuda si la necesitas.

## La historia de Git

**Git** es una creación de **Linus Torvalds**, el autor y primer desarrollador del kernel de Linux.

Linus Torvalds inició el desarrollo de Git en abril de 2005 cuando el sistema propietario de Gestión de Control de Fuente (SCM) utilizado para el desarrollo del kernel de Linux desde 2002, BitKeeper, revocó su licencia gratuita para el desarrollo de Linux.

![Linus Torvalds](/assets/git/Linus-Torvalds.jpg)

Torvalds quería un sistema distribuido que pudiera usar como BitKeeper, pero ninguno de los sistemas gratuitos disponibles cubría sus necesidades para el desarrollo del kernel de Linux.

El desarrollo de Git comenzó el 3 de abril de 2005 y el proyecto fue anunciado por Linus Torvalds el 6 de abril. 

Para el 29 de abril Torvalds había logrado sus objetivos de rendimiento, se registró al naciente Git grabando parches en el árbol del kernel de Linux a una velocidad de 6,7 parches por segundo. El 16 de junio, Git ya administraba la versión 2.6.12 del kernel.

Torvalds entregó el mantenimiento el 26 de julio de 2005 a Junio Hamano, uno de los principales contribuyentes al proyecto. Hamano fue responsable del lanzamiento 1.0 el 21 de diciembre de 2005.

Desde esa fecha y hasta la actualidad Git ha avanzado agregando funcionalidades, pero su esencia sigue siendo la misma desde sus primeros días.

## Requisitos previos para este curso

No necesitas saber nada sobre Git para leer este curso. Tampoco necesitas saber nada sobre la línea de comandos. Te explicaré todo desde el principio, incluidos todos los conceptos y comandos.

Necesitas una computadora con una terminal y un editor de texto.
En Windows puedes usar PowerShell o Git Bash. En Mac y Linux puedes utilizar la terminal integrada.

Puedes usar cualquier editor de texto que desees: VS Code, Sublime Text e incluso el Bloc de notas funcionarán.

## Lo que aprenderás en el curso

Aprenderás:

* Cómo configurar Git en tu computadora
* Cómo crear un repositorio
* Cómo confirmar cambios
* Cómo crear y fusionar ramas
* Cómo resolver conflictos de fusión
* Cómo usar repositorios remotos

En otras palabras, aprenderás a utilizar Git para las tareas cotidianas.

Por el momento no discutiremos temas avanzados como el rebase o los hooks de Git, aunque en la próxima actualización del curso llegaremos a estos temas avanzados. En lugar de ello, nos centraremos en construir una base sólida que te permitirá utilizar Git de forma eficaz.
